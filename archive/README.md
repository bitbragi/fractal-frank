# Archive

This folder holds documents Frank used to ship but no longer represents its current direction. They're kept here for historical context — please don't treat them as authoritative.

## Contents

### `SCRIBE_COVENANT_EDITION_SPEC.md`

The original "Scribe Protocol — Covenant Edition v1" spec. It describes a covenant-native rights system (Master / License / RevenueSplit) that Frank used to scaffold when it was branded specifically around the Fractal Bitcoin + Scribe Protocol use case.

This spec is **superseded**. The Bitamp project has carried these patterns forward on an independent Bitcoin Core v28 fork with new naming (`PID` instead of `SID`, `Bitamp*` covenant classes instead of `Scribe*`, consensus-level media rules instead of policy conventions). The current, maintained version lives in the Bitamp repo:

- **[`docs/BITAMP_COVENANT_EDITION_SPEC.md` (bitamp-testnet)](https://github.com/bitbragi/bitamp-testnet/blob/main/docs/BITAMP_COVENANT_EDITION_SPEC.md)**

Frank itself is now positioned as a **general-purpose OP_CAT + sCrypt AI instructor** for any Bitcoin fork or OP_CAT-enabled chain — Fractal, Bitamp, and whatever comes next. Frank no longer ships a chain-specific protocol; it ships the tools to build covenants on whichever chain you target.

If you're a historian or curious about how the design evolved, read on. If you're building today, follow the Bitamp link.
